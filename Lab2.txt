Функціональні вимоги
1. Система повинна розмежовувати права користувача та адміністратора.
2. Система повинна надавати можливість адміністратору авторизуватись під власним унікальним логіном та паролем
3. Система повинна надавати можливість адміністратору додавати, редагувати, видаляти та архівувати  записи.
4. Система повинна надавати можливість адміністратору додавати медіа-дані: фото, відео, файли.
5. Система повинна надавати можливість адміністратору затверджувати та видаляти  коментарі.
6. Система повинна надавати користувачу можливість перегляду всіх записів.
7. Система повинна надавати користувачу можливість перегляду окремого запису.
8. Система повинна надавати користувачу можливість коментування запису.
9. Система повинна надавати користувачу можливість ввести ім'я та e-mail, щоб прокоментувати запис.
10. Система повинна надавати користувачу можливість перегляду всіх коментарів запису.
11. Система повинна надавати користувачу можливість пошуку записів.


Нефункціональні вимоги
Зручність використання
1. Інтерфейс не повинен вимагати додаткової підготовки користувачів.
Надійність
1. Для користувачів система повинна бути доступна 99.9% часу.
Продуктивність
1. Система повинна бути здатна підтримувати мінімум 1000 одночасно працюючих користувачів.
2. Час відгуку для типових завдань повинен бути менше 3 секунд.
Обмеження проектування
1. Система повинна задовольняти вимогам роботи браузерів:
* Mozilla Firefox (з версії 49.0)
* Google Chrome (з версії 50.0)
* Internet Explorer (з 10.0 версії)
* Opera (з версії 41.0.2353)
* Microsoft Edge (з версії 33.0)
* Safari (з версії 10.1.2)
1. Система повинна бути адаптивною, тобто коректно працювати на мобільних пристроях з такими ОС:
* Android (з версії 5.1.х)
* iOS (з версії 10.0.1)
Масштабованість і конфігурація
1. Система повинна відповідати модульній структурі.
2. Система повинна бути здатна до розширення та масштабування.
